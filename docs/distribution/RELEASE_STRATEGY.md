# Project Nimbus Release Strategy

## Philosophy

Project Nimbus was built with an **accessibility-first philosophy**. The goal is to make gaming and joystick-based applications accessible to people with disabilities who cannot use traditional physical controllers. This mission drives every decision about how the software is released and sustained.

### Core Values
- **Free for end users** — People with disabilities should never face a paywall to access assistive technology
- **Open source** — Transparency builds trust, enables community contributions, and allows customization for individual needs
- **Sustainable development** — The creator deserves compensation for their time and expertise
- **Community-driven** — Users with disabilities are the experts on their own needs

## Licensing Model

### Current License: MIT

Project Nimbus is released under the **MIT License**, which allows:
- ✅ Free use for any purpose (personal, commercial, educational)
- ✅ Modification and redistribution
- ✅ Integration into other projects
- ✅ Commercial use by organizations

This license was chosen because:
1. **No barriers** — Anyone can download and use the software immediately
2. **Corporate adoption** — Hospitals, rehab centers, and gaming companies can integrate it freely
3. **Community contributions** — Developers can fork, improve, and contribute back
4. **Trust** — Open source builds credibility in the assistive technology community

### Alternative Licensing Considerations

| License | Pros | Cons |
|---------|------|------|
| **MIT** (current) | Maximum adoption, corporate-friendly | No copyleft protection |
| **GPL v3** | Ensures derivatives stay open source | May deter corporate integration |
| **Apache 2.0** | Patent protection, corporate-friendly | More complex than MIT |
| **Dual License** | Commercial license for revenue | Complexity, may limit adoption |

**Recommendation**: Keep MIT for maximum impact. Revenue should come from sponsorships and grants, not licensing restrictions.

## Recommended Strategy

### Phase 1: Foundation (Now)

1. **Document everything** — User stories, impact metrics, testimonials
2. **Set up GitHub Sponsors** — Low effort, immediate availability
3. **Build relationships** — Connect with AbleGamers, SpecialEffect, etc.
4. **Create a simple website** — Landing page with mission, downloads, sponsor info

### Phase 2: Outreach (3-6 months)

1. **Apply for grants** — Start with AbleGamers, Craig H. Neilsen Foundation
2. **Approach corporate sponsors** — Microsoft, Logitech, Razer
3. **Partner with institutions** — Offer free pilots to hospitals/rehab centers
4. **Present at conferences** — GDC Accessibility Summit, CSUN, ATIA

### Phase 3: Sustainability (6-12 months)

1. **Establish recurring sponsorships** — Annual commitments from 2-3 sponsors
2. **Consider 501(c)(3)** — Nonprofit status for tax-deductible donations
3. **Build a small team** — Use funding for part-time contributors
4. **Expand features** — Based on sponsor and user feedback

## Summary

**The recommended approach:**

1. **Keep it free and open source (MIT)** — Maximize impact and adoption
2. **Pursue corporate sponsorships** — Primary revenue source
3. **Apply for accessibility grants** — Supplemental funding
4. **Build institutional partnerships** — Credibility and user feedback
5. **Enable community funding** — GitHub Sponsors, Open Collective
6. **Offer optional services** — Consulting, training, custom work

This model allows Project Nimbus to remain free for everyone who needs it while providing sustainable income for continued development.
